1/*
2 * This file contains the Genetic Algorithm for finding anonymized tables.
3 * Like MinGen (IE Brute-Force), a Genetic Algorithm is not so much like Wireguard,
4 * where the algorithm has an explicit set of steps and methods, but more a general
5 * approach that is customized on a case-by-case basis. The source that I consulted was:
6 * https://books.google.com/books?id=GlGpDgAAQBAJ&dq=genetic+algorithm+evolutionary+algorithms&pg=PP2
7 * but also like MinGen, you don't need to read it to understand the contents of this file.
8 * Generally, a Genetic Algorithm consists of a few key, and broad, steps:
9 * 1. Create an initial population
10 * 2. Sort this population based on a fitness metric.
11 * 3. Choose the best of the population, and generate a new population from these top results.
12 * 4. Continue step 2-4 until an end condition is met, such as a certain score, or generation cap.
13 *
14 * For the MinGen implementation, the process was rather straightforward: the brute-force algorithm was
15 * implemented, and then optimizations were added on top to speed up the calculations. Perhaps fittingly,
16 * the development of the Genetic Algorithm took a more gradual, iterative approach, as I tried different
17 * strategies.
18 *
19 * For this problem in particular, there are two key values that we want to express in a fitness score:
20 * 1. The K-Anonymity
21 * 2. The Scoring Function (IE Minimal-Distortion or Certainty)
22 * With the latter, adding this into a fitness function is quite easy. Lower scores are better, so simply
23 * use it as a denominator (We need to be weary of tables that don't make any changes, causing a /0)
24 * K-Anonymity, however, is tricky: higher K-Score is better, but only to a point, and lower K-Score is
25 * absolutely unacceptable for the final table. If we simply used K-Score as a numerator, the algorithm would
26 * start valuing higher K, despite the user not needing more than the value they provided, and we'd get completely suppressed
27 * tables to maximize K. For this, we can simply add a cap such that K values higher than the requested give no additional
28 * fitness than getting the score exactly. However, the second issue is quite difficult to deal with: we should never
29 * return a table that doesn't meet the requested K-Anonymity, but how can we enforce such a requirement in a formula?
30 *
31 * The first approach was simply giving K-Anonymity a high weight, and unsurprisingly,
32 * the results were poor; the algorithm would never actually return a K-Anonymous table, with averages hovering around 1-1.75
33 * (Target = 2). The problem was that in the search-space, K-Anonymous tables are exceedingly rare, and not even excessive
34 * mutation rates (Which is essentially just stochastic) could find them.
35 *
36 * The next approach was simply removing K-Anonymity from the equation entirely: pull the initial population as a set of random,
37 * K-Anonymous tables (We could use our MinGen implementation to simply iterate until it finds such a table, regardless of score),
38 * and enforce mutations/inheritance such that the children are also K-Anonymous. Unfortunately, while this did produce results,
39 * it was terribly slow. Since we were tying our initial generation to MinGen (Even if not a exhaustive search), we weren't getting
40 * much faster than just using MinGen, which was the chief reason of this implementation. Enforcing K-Anonymity at each change also
41 * prevented the algorithm from gradual evolution; say table S has a fitness of 15, and S' has a value of 20 (Higher is better for fitness)
42 * but to reach S' we would need to make a series of changes to which the intermediaries would not be K-Anonymous. The only way for
43 * this implementation to do this would be if it was exceedingly lucky and mutated all the changes in one pass, which constrained
44 * evolution.
45 *
46 * Finally, and the implementation that exists below, was to simply run the algorithm in two passes (Originally, it was quite literally
47 * two simulations with separate scoring functions, but has since been optimized into just a two-stage scoring function). The first
48 * step is simply reaching K-Anonymity. If the Table isn't K-Anonymous, its score is simply the average K-Anonymity for all rows. It's only
49 * once it's reached K-Anonymity that the S-Score function is added on top of that. This implementation works precisely as expected:
50 * we can feed entirely random tables as the initial generation, do not need to enforce any requirements on changes and mutations, which
51 * lets mutations run wild, and get very good (Although not necesssarily the best) results. The benchmark for this algorithm was table2.csv
52 * in `examples`, which contains more states than C++ is capable of printing (Hence far out of reach for MinGen). This implementation, coupled
53 * with optimizations via caching, can produce convincing tables for this input.
54 *
55 */

73 // The instance is just a table and its fitness.
74 // We want to store our Tables in a std::set, because then we don't actually
75 // need to sort our Tables, we can let the Standard Library do it for us as we emplace.

91 // Bonuses, and the current best. K is a float because we return the average K across each rows.
92 float k = 2.0, best = INFINITY;
93
94 // The metric to use.
95 metric::metric m = metric::md;
96
97 // The generation limit, the size of each generation, states encountered, the amount of tables deemed
98 // the best, the rate of mutations, amount of cells in the table, and current iteration.
99 size_t max = 1000, population = 100, states = 0, cutoff = 10, m_rate = 10, cells = 0, iter = 0;
100
101 // Random number generation, one for deciding who a table should partner with, another for mutations.
102 // Mutation uses a simple scheme, where we roll a number from 0-100, and add our mutation rate on top
103 // of that. Any value <= 50 we use the value from the first, any value > 50 <= 100 we use the value from
104 // the second, and any value > 100 is a mutation. A mutation is just choosing at random one of the
105 // possible ways a cell can be changed given Domains/Ranges; it's the same method that MinGen used to
106 // iterate over all possibilities, it just shuffles them and we draw the top (The randomness was used
107 // by the non-exhaustive MinGen so it didn't just return the same values over and over again).
108 //
109 // With a set of discrete cells, this mimics genetic recombination to a tee; the
110 // result is that about half the cells are inherited from one parent,
111 // the other half from the other, with a bit of mutations sprinkled in.
112 std::uniform_int_distribution<> roll, mutations = std::uniform_int_distribution<>(0, 100 + m_rate);
113
114 // Lets the main thread display what the worker is working on.
115 instance view;
116 std::mutex lock;
117
118 // Using a multiset here makes things a lot easier for us, since it already manages sorting.
119 // We can simply initialize a set, and populate it with the new generation and it will be
120 // automatically ordered, letting us pop from the front for whatever percentile we're looking for.
121 // A multiset is necessary because we want non-unique values.
122 std::multiset<instance, decltype(score_lambda)> generation = {};

154 inline float fitness(const table::Table& f) const {
155
156 // If our table is K-Anonymous, then add our s_score to the final score.
157 if (metric::k_anonymity(f, original, k)) {
158
159 // The fitness takes our k_value; not the average, so a score greater than k just adds k to the score and prevents
160 // the algorithm from liking excessive k-values when the user doesn't need it.
161 float score = k;
162
163 // S_Score will be, at most, 1, which means that simply dividing as-is would actually lead to this lowering fitness. To fix,
164 // we just multiply it by a fixed value, in this case just the number of cells. There's no mathematical reason for this other
165 // than giving K-Anonymous scores a boost.
166 auto s_score = (m == metric::md ? metric::minimal_distortion(f, original, INFINITY) : metric::certainty_score(f, original, INFINITY));
167 return ((score * cells) / s_score);
168 }
169
170 // Otherwise, the score is the average K-Anonymity for each row.
171 else return metric::av_k_anonymity(f, original) / k;
172 }

183 void combine(table::Table& first, const table::Table& second) {
184 for (size_t col = 0; col < original.columns(); ++col) {
185 auto& fc = first.get_column(col); const auto& sc = second.get_column(col);
186
187 // Don't try and mutate non-quasi values.
188 if (fc.sensitivity != table::quasi) continue;
189
190 for (size_t row = 0; row < fc.data.size(); ++row) {
191
192 // Roll for what to do. The range is 0-100 + mutation
193 // If we roll above 100, its a mutation
194 // Otherwise < 50 borrow, >= 50 keep.
195 auto mutate = mutations(shared::gen);
196
197 // If we roll a mutation, get all possible values, and just choose one at random (The true in the mutations() call).
198 if (mutate > 100) fc.data[row] = original.mutations(original.index(col).data[row], original.get_column(col), true)[0];
199
200 // If it's less than 50, we take from the other table, otherwise we "take" from the first by not doing anything,
201 // since we're modifying it in place.
202 else if (mutate < 50) fc.data[row] = sc.data[row];
203 }
204 }
205 }

212 std::multiset<table::Table> anonymize_worker() {
213
214 // Lambda to get the best of the generation, emplace into a multiset, and return it.
215 auto generate_return = [](const std::multiset<instance, decltype(score_lambda)>& gen) {
216 std::multiset<table::Table> ret;
217 float best = gen.begin()->s;
218 for (const auto& solution : gen) {
219 if (solution.s != best) break;
220 ret.emplace(solution.table);
221 }
222 return ret;
223 };
224
225
226 // We want to gradually increase mutation rate.
227 size_t tenth = max/10;
228
229 // Iterate for max generations.
230 for (iter = 0; iter < max; ++iter) {
231
232 if ((iter + 1) % tenth == 0) {
233 // This is probably one of the best features of this algorithm, and I don't know entirely
234 // how standard it is. Basically, it gradually increases the mutation rate over time,
235 // such that the Genetic Algorithm, to which beneficial traits are passed down by previous generations,
236 // devolves into a largely stochastic algorithm (With the default m_rate of 10, the last 10% will
237 // have a mutation rate of 3840:100). This ONLY works because we keep the best of the previous
238 // generation, which means crazy mutations that tank score will just be left behind for the next
239 // generation, but it allows us to break out of local-maximums by giving tables the ability to
240 // pretty much re-roll every cell to something random.
241 m_rate *= 2;
242 std::uniform_int_distribution<> roll, mutations = std::uniform_int_distribution<>(0, 100 + m_rate);
243
244 update_view(*generation.begin());
245 }
246
247 best = generation.begin()->s;
248
249 // The next generation.
250 std::multiset<instance, decltype(score_lambda)> children = {};
251
252 // Cutoff determimes the top scores to keep, but we need to preserve the same generation size, so this determines
253 // how many children each of the best need to produce, on top of including itself.
254 size_t offspring = (population - cutoff)/cutoff;
255
256 // Get the top score.
257 for (size_t x = 0; x < cutoff; ++x) {
258
259 // Get our current, put them into the next generation.
260 auto current = *std::next(generation.begin(), x);
261 children.emplace(current);
262 ++states;
263
264 // Make offspring.
265 for (size_t y = 0; y < offspring; ++y) {
266 ++states;
267
268 // Get a partner from the top n; this can be itself.
269 const auto& partner = *std::next(generation.begin(), roll(shared::gen));
270
271 // Make a child
272 combine(current.table, partner.table);
273 current.s = fitness(current.table);
274 children.emplace(current);
275 }
276 }
277
278 // Replace
279 generation = children;
280 }
281
282 // Once we're done if iterations, return the best.
283 return generate_return(generation);
284 }

305 void anonymize(
306 const size_t& k_val,
307 const metric::metric& m_val,
308 const size_t& iters,
309 const size_t& p_val,
310 const size_t& mut_rate
311 ) {
312
313 // Setup our values.
314 k = k_val;
315 m = m_val;
316 if (iters != SIZE_MAX) max = iters;
317 population = p_val;
318 m_rate = mut_rate;
319
320 // Make random tables.
321 log("Generating Random Tables");
322 for (size_t x = 0; x < population; ++x) {
323 auto i = original.random();
324 auto s = fitness(i);
325 generation.emplace(instance{.s=s, .table=i});
326 };
327 roll = std::uniform_int_distribution<>(0, cutoff);
328
329 // Some helpful info.
330 log("Scoring Metric: " << m);
331 auto total = original.get_distinct();
332 if (total == SIZE_MAX) {log("There are more states than your computer is capable of storing and displaying. Good luck.");}
333 else log("Possible Configurations: " << total);
334 if (max != -1) log("Non-Exhastive Search: Results may not be best");
335
336 shared::pick_mode();
337
338 // Anonymize.
339 auto start = std::chrono::steady_clock::now();
340
341 std::multiset<table::Table> tables;
342 if (shared::single_thread) tables = anonymize_worker();
343 else {
344
345 // Since when has C++ had async? To make it better, the GCC/Clang version uses a Thread Pool, which means this
346 // costs us nothing to spin up, and with nice status functions to wait until its finished. This is why C++ is the best.
347 std::future<std::multiset<table::Table>> future = std::async(std::launch::async, &GeneticAlgorithm::anonymize_worker, this);
348
349 update_view(*generation.begin());
350 auto sample = get_view();
351 sample.table.update_widths();
352
353 // For a smooth 60 FPS ;)
354 for (size_t x = 0; future.wait_for(std::chrono::milliseconds(17)) != std::future_status::ready; ++x) {
355 shared::clear();
356
357 std::cout << shared::print_title(shared::ga, x/5) << std::endl;
358
359 if (sample.s != best) {
360 sample = get_view();
361 sample.table.update_widths();
362 }
363
364 // Print some statistics.
365 std::cout << "Generation: " << iter << "/" << max << std::endl;
366 std::cout << "Fitness (Larger is better): " << sample.s << std::endl;
367
368 // Print the current best table
369 //sample->table.update_widths();
370 sample.table.print();
371 }
372 tables = future.get();
373 }
374
375 auto end = std::chrono::steady_clock::now();
376 auto duration = end - start;
377
378 const auto& b = *tables.begin();
379 best = (m == metric::md ? metric::minimal_distortion(b, original, INFINITY) : metric::certainty_score(b, original, INFINITY));
380 if (!metric::k_anonymity(b, original, k)) {
381 std::cout << "WARNING: Result is not k-anonymous! Increase iterations or population size!" << std::endl;
382 getchar();
383 }
384
385 // Print some stats, and reset for another call.
386 metric::print_stats(tables, max, total, states, best, duration);
387 reset();
388 }
389 };
